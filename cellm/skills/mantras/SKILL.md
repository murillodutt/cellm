# Work Mantras — Before Planning, Before Writing

Three non-negotiable principles. Apply BEFORE planning and BEFORE writing code. Every time.

| Mantra | Meaning | Anti-Pattern |
|--------|---------|-------------|
| VERIFY DEEPLY | Explore what already exists in detail. Never propose anything that duplicates or ignores existing work. Question everything. Think out loud. | Proposing new code without searching for existing implementations, skipping codebase exploration |
| BEST PATH | Not the easy path — the most solid, structured, professional choice for the medium/long term. | "It works" instead of "it's right", shortcuts that create tech debt, settling for quick fixes |
| DOCS FIRST | Always check current documentation, even when confident. An update or correction may completely change what you're about to write. | Generating from training data, skipping MCP doc tools, assuming versions |

## NEVER

- **Propose without exploring** — search codebase first, read existing code, verify no duplication
- **Choose easy over right** — every decision optimizes for medium/long term
- **Trust memory over docs** — always verify against current sources before writing
