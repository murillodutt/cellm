# Design Thinking — Before Code

Commit to a direction before writing a single line:

1. **Purpose** — What problem does this interface solve? Who uses it?
2. **Tone** — Pick ONE and commit: brutally minimal, maximalist, retro-futuristic, organic, luxury, playful, editorial, brutalist, art deco, soft/pastel, industrial. Use as inspiration, not constraint.
3. **Differentiator** — What makes this unforgettable? One thing someone remembers.
4. **Stack** — No framework specified? Single page → HTML. Stateful app → React/Vue.

## Aesthetic Axes

| Axis | Instruction |
|------|-------------|
| Typography | Distinctive display + refined body. Never Inter, Roboto, Arial, system fonts. |
| Color | Dominant + sharp accent. CSS variables. Never timid evenly-distributed palettes. |
| Motion | One orchestrated moment > scattered micro-interactions. Staggered reveals, scroll-triggered, surprising hovers. CSS-only for HTML, Motion library for React. |
| Composition | Asymmetry. Overlap. Diagonal flow. Grid-breaking. Generous negative space OR controlled density. |
| Atmosphere | Gradient meshes, noise textures, geometric patterns, layered transparencies, grain overlays. Never flat solid backgrounds. |

Match complexity to vision: maximalist needs elaborate code; minimalist needs precise spacing and typography.

## Self-Check

Before delivering, verify:
- [ ] Would I put this in a portfolio?
- [ ] Is the tone consistent across every element?
- [ ] Can I point to ONE generic element? If yes, redesign it.

## NEVER

- **Inter, Roboto, Arial, system fonts** — pick characterful fonts
- **Purple gradients on white** — the universal AI slop signature
- **Predictable layouts** — if it looks like a template, it is one
- **Converge across generations** — vary themes, fonts, aesthetics every time
- **Motivational filler** — every token must carry signal
